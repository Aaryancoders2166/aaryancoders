CSS is the language that defines the presentation of a web page. It is used to add color, background images, and textures, and to arrange elements on the page. However, CSS does a lot more than just paint a pretty picture. It is also used to enhance the usability of a website. The image below shows the front page of YouTube. On the left, is a regular rendering of the page, and on the right you can see how it looks without CSS.<\/p>\n

<\/p>\n

What is CSS?<\/span><\/h2>\n

CSS stands for Cascading Style Sheets and it is the language used to style the visual presentation of web pages. CSS is the language that tells web browsers how to render the different parts of a web page.<\/p>\n

Every item or element on a web page is part of a document written in a markup language. In most cases, HTML is the markup language, but there are other languages in use such as XML. Throughout this guide we will use HTML to demonstrate CSS in action, just keep in mind that the same principles and techniques apply if you\u2019re working with XML or a different markup language.<\/p>\n

How is CSS Different From HTML?<\/span><\/h3>\n

The first thing to understand when approaching the topic of CSS is when to use a styling language like CSS and when to use a markup language such as HTML.<\/p>\n

    \n
  • All critical website content<\/em> should be added to the website using a markup language such as HTML.<\/li>\n
  • Presentation<\/em> of the website content should be defined by a styling language such as CSS.<\/li>\n<\/ul>\n

    Blog posts, page headings, video, audio, and pictures that are not part of the web page presentation should all be added to the web page with HTML. Background images and colors, borders, font size, typography, and the position of items on a web page should all be defined by CSS.<\/p>\n

    It\u2019s important to make this distinction because failing to use the right language can make it difficult to make changes to the website in the future and create accessibility and usability issues for website visitors using a text-only browser or screen reader.<\/p>\n

    CSS Syntax<\/span><\/h2>\n

    CSS syntax includes selectors, properties, values, declarations, declaration blocks, rulesets, at-rules, and statements.<\/p>\n

      \n
    • A selector<\/dfn> is a code snippet used to identify the web page element or elements that are to be affected by the styles.<\/li>\n
    • A property<\/dfn> is the aspect of the element that is to be affected. For example, color, padding, margin, and background are some of the most commonly used CSS properties.<\/li>\n
    • A value<\/dfn> is used to define a property<\/em>. For example, the property color might be given the value of red like this: color: red;<\/code>.<\/li>\n
    • The combination of a property<\/em> and a value<\/em> is called a declaration<\/dfn>.<\/li>\n
    • In many cases, multiple declarations<\/em> are applied to a single selector<\/em>. A declaration block<\/dfn> is the term used to refer to all of the declarations applied to a single selector<\/em>.<\/li>\n
    • A single selector<\/em> and the declaration block<\/em> that follows it in combination are referred to as a ruleset<\/dfn>.<\/li>\n
    • At-rules<\/dfn> are similar to rulesets<\/em> but begin with the @<\/em> sign rather than with a selector<\/em>. The most common at-rule is the @media<\/code> rule which is often used to create a block of CSS rules that are applied based on the size of the device viewing the web page.<\/li>\n
    • Both rulesets<\/em> and at-rules<\/em> are CSS statements<\/dfn>.<\/li>\n<\/ul>\n

      An Example of CSS Syntax<\/span><\/h3>\n

      Let\u2019s use a block of CSS to clarify what each of these items is.<\/p>\n

      h1 {\n    color: red;\n    font-size: 3em;\n    text-decoration: underline;\n    }\n<\/code><\/pre>\n
      In this example, h1<\/code> is the selector. The selector is followed by a declaration block that includes three declarations. Each declaration is separated from the next by a semicolon. The tabs and line breaks are optional but used by most developers to make the CSS code more human-readable.<\/p>\n
      By using h1<\/code> as the selector, we are saying that every level 1 heading on the web page should follow the declarations contained in this ruleset.<\/p>\n
      The ruleset contains three declarations:<\/p>\n
        \n
      • color:red;<\/code><\/li>\n
      • font-size: 3em;<\/code><\/li>\n
      • text-decoration: underline;<\/code><\/li>\n<\/ul>\n
        color<\/code>, font-size<\/code>, and text-decoration<\/code> are all properties. There are literally hundreds of CSS properties you can use, but only a few dozen are commonly used.<\/p>\n
        We applied the values red<\/code>, 3em<\/code>, and underline<\/code> to the properties we used. Each CSS property is defined to accept values formatted in a specific way.<\/p>\n
        For the color<\/code> property we can either use a color keyword or a color formula in Hex, RGB, or HSL format. In this case, we used the color keyword red<\/code>. There are a few dozen color keywords available in CSS3, but millions of colors can be accessed with the other color models.<\/p>\n
        We applied the value of 3em<\/code> to the property font-size<\/code>. There are a wide range of size units we could have used including pixels, percentages, and more.<\/p>\n
        Finally, we added the value underline<\/code> to the property text-decoration<\/code>. We could have also used overline<\/code> or line-through<\/code> as values for text-decoration<\/code>. In addition, CSS3 allows for the use of the line-styles solid, double, dotted, dashed, and wavy was well the specification of text-decoration colors. We could have applied all three values at once by using a declaration like this:<\/p>\n
        text-decoration: blue double underline;\n<\/code><\/pre>\n
        That rule would cause the h1<\/code> in our initial example to be underlined with a blue double line. The text itself would remain red as defined in our color<\/code> property.<\/p>\n

        Preparing HTML Markup for Styling<\/span><\/h2>\n
        CSS should be used to add content to a web page. That task is best handled by markup languages such as HTML and XML. Instead, CSS is used to pick items that already exist on a web page and to define how each item should appear.<\/p>\n
        In order to make it as easy as possible to select items on a web page, identifiers should be added to elements on the webpage. These identifiers, often called hooks in the context of CSS, make it easier to identify the items that should be affected by the CSS rules.<\/p>\n
        Classes and IDs are used as hooks for CSS styles. While the way CSS renders is not affected by the use of classes and hooks, they give developers the ability to pinpoint HTML elements that they wish to style.<\/p>\n
        Classes and IDs aren\u2019t interchangeable. It\u2019s important to know when to use each.<\/p>\n
        When to Use Classes<\/span><\/h3>\n
        Use classes when there are multiple elements on a single web page that need to be styled. For example, let\u2019s say that you want links in the page header and footer to be styled in a consistent manner, but not the same way as the links in the body of the page. To pinpoint those links you could add a class to each of those links or the container holding the links. Then, you could specify the styles using the class and be sure that they would only be applied to the links with that class attribute.<\/p>\n
        When to Use IDs<\/span><\/h3>\n
        Use IDs for elements that only appear once on a web page. For example, if you\u2019re using an HTML unordered list for your site navigation, you could use an ID such as nav<\/em> to create a unique hook for that list.<\/p>\n
        Here\u2019s an example of the HTML and CSS code for a simple navigation bar for a basic e-commerce site.<\/p>\n
        <style>\n    #nav {\n        background: lightgray;  \n        overflow: auto; \n        }\n    #nav li {\n        float: left;\n        padding: 10px;\n        }\n    #nav li:hover {\n        background: gray;\n        }\n<\/style>\n\n<ul id=\"nav\">\n    <li><a href=\"\">Home<\/a><\/li>\n    <li><a href=\"\">Shop<\/a><\/li>\n    <li><a href=\"\">About Us<\/a><\/li>\n    <li><a href=\"\">Contact Us<\/a><\/li>\n<\/ul>\n<\/code><\/pre>\n
        That code would produce a horizontal navigation menu with a light gray background beginning from the left-hand side of the page. Each navigation item will have 10 pixels of spacing on all sides and the background of each item will become darker when you allow your mouse to hover over it.<\/p>\n
        Any additional lists on the same web page would not be affected by that code.<\/p>\n
        \n
        #example-nav {
        background: lightgray;
        overflow: auto;
        }
        #example-nav li {
        float: left;
        padding: 10px;
        }
        #example-nav li:hover {
        background: gray;
        }<\/p>\n
          \n
        • Home<\/li>\n
        • Shop<\/li>\n
        • About Us<\/li>\n
        • Contact Us<\/li>\n<\/ul>\n<\/div>\n
          When Not to Use Hooks<\/span><\/h3>\n
          You don\u2019t have to add a class or ID to an HTML element in order to style it with CSS. If you know you want to style every instance of a specific element on a web page you can use the element tag itself.<\/p>\n
          For example, let\u2019s say that you want to create consistent heading styles. Rather than adding a class or ID to each heading it would be much easier to simply style all heading elements by using the heading tag.<\/p>\n
          <style>\n    ul {\n        list-style-type: upper-roman;\n        margin-left: 50px;\n        }\n    p {\n        color: darkblue\n        }\n<\/style>\n\n<p>Some paragraph text here. Two short sentences.<\/p>\n<ul>\n<li>A quick list<\/li>\n<li>Just two items<\/li>\n<\/ul>\n<p>Additional paragraph text here. This time let's go for three sentences. Like this.<\/p>\n<\/code><\/pre>\n
          That code would render like this.<\/p>\n
          \n
          .code_sample ul {
          list-style-type: upper-roman;
          margin-left: 50px;
          }
          .code_sample p {
          color: darkblue
          }<\/p>\n
          \n
          Some paragraph text here. Two short sentences.<\/p>\n
            \n
          • A quick list<\/li>\n
          • Just two items<\/li>\n<\/ul>\n
            Additional paragraph text here. This time let\u2019s go for three sentences. Like this.<\/p>\n
              \n
            • Another list<\/li>\n
            • Still just two items<\/li>\n<\/ul>\n<\/div>\n<\/div>\n
              In this case, even though we only wrote the style rules for ul<\/code> and p<\/code> elements once each, multiple items were affected. Using element selectors is a great way to create an attractive, readable, and consistent website experience by creating consistent styling of headings, lists, and paragraph text on every page of the website.<\/p>\n
              Best Practices for Preparing Your Markup for Styling<\/span><\/h3>\n
              Now that you know how classes, IDs, and element tags can be used as hooks for CSS rulesets, how can you best implement this knowledge to write markup that makes it easy to poinpoint specific elements?<\/p>\n
                \n
              • Apply classes liberally and consistently. Use classes for items that should be aligned in one direction or the other, and for any elements that appear repeatedly on a single web page.<\/li>\n
              • Apply IDs to items that appear only once on a web page. For example, use an ID on the div<\/code> that contains your web page content, on the ul<\/code> that that contains the navigation menu, and on the div<\/code> that contains your web page header.<\/li>\n<\/ul>\n
                Ways of Linking CSS Rules to an HTML Document<\/span><\/h2>\n
                There are three ways of adding CSS rules to a web page:<\/p>\n
                  \n
                • Inline styles<\/li>\n
                • Internal stylesheets<\/li>\n
                • External stylesheets<\/li>\n<\/ul>\n
                  In the vast majority of cases, external stylesheets should be used. However, there are instances where inline styles or internal stylesheets may be used.<\/p>\n
                  Inline Styles<\/span><\/h3>\n
                  Inline styles are applied to specific HTML elements. The HTML attribute style<\/code> is used to define rules that only apply to that specific element. Here\u2019s a look at the syntax for writing inline styles.<\/p>\n
                  <h1 style=\"color:red; padding:10px; text-decoration:underline;\">Example Heading<\/h1>\n<\/code><\/pre>\n
                  That code would cause just that heading to render with red underlined text and 10 pixels of padding on all sides. There are very few instances where inline styles should be used. In nearly all cases they should be avoided and the styles added to a stylesheet.<\/p>\n
                  Internal Stylesheets<\/span><\/h3>\n
                  The earlier examples in this tutorial make use of internal stylesheets. An internal stylesheet is a block of CSS added to an HTML document head<\/code> element. The style<\/code> element is used between the opening and closing head<\/code> tags, and all CSS declarations are added between the style<\/code> tags.<\/p>\n
                  We can duplicate the inline styles from the code above in an internal stylesheet using this syntax.<\/p>\n
                  <head>\n    <style>\n        h1 {\n            color: red;\n            padding: 10px;\n            text-decoration: underline;\n            }\n    <\/style>\n<\/head>\n<body>\n    <h1>Example Heading<\/h1>\n<\/body>\n<\/code><\/pre>\n
                  That code would produce the same results as the inline styles. However, the benefit to using internal stylesheets rather than inline styles is that all h1<\/code> elements on the page will be affected by the styles.<\/p>\n
                  External Stylesheets<\/span><\/h3>\n
                  External stylesheets are documents containing nothing other than CSS statements. The rules defined in the document are linked to one or more HTML documents by using the link<\/code> tag within the head<\/code> element of the HTML document.<\/p>\n
                  To use an external stylesheet, first create the CSS document.<\/p>\n
                  \/*************************************************\nSave with a name ending in .css such as styles.css\n*************************************************\/\nh1 {\n    color: red;\n    padding: 10px;\n    text-decoration: underline;\n    }\n<\/code><\/pre>\n
                  Now that we have an external stylesheet with some styles, we can link it to an HTML document using the link<\/code> element.<\/p>\n
                  <head>\n<link rel=\"stylesheet\" type=\"text\/css\" href=\"styles.css\">\n<\/head>\n<body>\n    <h1>Example Heading<\/h1>\n<\/body>\n<\/code><\/pre>\n
                  When this HTML document is loaded the link<\/code> tag will cause the styles in the file styles.css<\/span><\/em> to be loaded into the web page. As a result, all level 1 heading elements will appear with red text, underlined, and with 10 pixels of padding applied to every side.<\/p>\n
                  When to Use Each Method<\/span><\/h3>\n
                  In nearly all cases external stylesheets are the proper way to style web pages. The primary benefit to using external stylesheets is that they can be linked to any number of HTML documents. As a result, a single external stylesheet can be used to define the presentation of an entire website.<\/p>\n
                  Internal stylesheets may be used when designing a simple one-page website. If the website will never grow beyond that single initial page using an internal stylesheet is acceptable.<\/p>\n
                  Inline styles are acceptable to use in two instances:<\/p>\n
                    \n
                  1. When writing CSS rules that will only be applied to a single element on a single web page.<\/li>\n
                  2. When applied by a WYSIWYG editor such as the tinyMCE editor integrated into a content management system such as WordPress.<\/li>\n<\/ol>\n
                    In all other cases, inline styles should be avoided in favor of external stylesheets.<\/p>\n
                    How CSS Works<\/span><\/h2>\n
                    When writing CSS, there are many times that rules are written that conflict with each other. For example, when styling headers, all of the following rules may apply to an h1<\/code> element.<\/p>\n
                      \n
                    • An element-level rule creating consistent h1<\/code> rendering across all pages of the website.<\/li>\n
                    • A class-level rule defining the rendering of h1<\/code> elements occurring in specific locations \u2013 such as the titles of blog posts.<\/li>\n
                    • An id-level element defining the rendering of an h1<\/code> element used in just one place on a one or more web pages \u2013 such as the website name.<\/li>\n<\/ul>\n
                      How can a developer write rules that are general enough to cover every h1<\/code> yet specific enough to define styles that should only appear on specific instances of a given element?<\/p>\n
                      CSS styles follow two rules that you need to understand to write effective CSS. Understanding these rules will help you write CSS that is broad when you need it to be, yet highly-specific when you need it to be.<\/p>\n
                      The two rules that govern how CSS behaves are inheritance and specificity.<\/p>\n
                      Cascading Inheritance<\/span><\/h3>\n
                      Why are CSS styles called cascading<\/em>? When multiple rules are written that conflict with each other, the last rule written will be implemented. In this way, styles cascade downward and the last rule written is applied.<\/p>\n
                      Let\u2019s look at an example. Let\u2019s write two CSS rules in an internal stylesheet that directly contradict each other.<\/p>\n
                      <head>\n    <style>\n        p {color: red;}\n        p {color: blue;}\n    <\/style>\n<\/head>\n<body>\n<p>What color will the text of this paragraph be?<\/p>\n<\/body>\n<\/code><\/pre>\n
                      The browser will cascade through the styles and apply the last style encountered, overruling all previous styles. As a result, the heading is blue.<\/p>\n
                      \n
                      .code_sample_p {color: red;}
                      .code_sample_p {color: blue;}<\/p>\n
                      What color will the text of this paragraph be?<\/p>\n<\/div>\n
                      This same cascading effect comes into play when using external stylesheets. It\u2019s common for multiple external stylsheets to be used. When this happens, the style sheets are loaded in the order they appear in the HTML document head<\/code> element. Where conflicts between stylesheet rules occur, the CSS rules contained in each stylesheet will overrule those contained in previously loaded stylesheets. Take the following code for example:<\/p>\n
                      <head>\n<link rel=\"stylesheet\" type=\"text\/css\" href=\"styles_1.css\">\n<link rel=\"stylesheet\" type=\"text\/css\" href=\"styles_2.css\">\n<\/head>\n<\/code><\/pre>\n
                      The rules in styles_2.css<\/em> will be applied if there are conflicts between the styles contained in these two stylesheets.<\/p>\n
                      Inheritance of styles is another example of the cascading behavior of CSS styles.<\/p>\n
                      When you define a style for a parent element the child elements receive the same styling. For example, if we apply color styling to an unordered list, the child list items will display the same styles.<\/p>\n
                      <head>\n    <style>\n        ul {color: red;}\n    <\/style>\n<\/head>\n<body>\n    <ul>\n        <li>Item 1<\/li>\n        <li>Item 2<\/li>\n    <\/ul>\n<\/body>\n<\/code><\/pre>\n
                      Here\u2019s how that code would render.<\/p>\n
                      \n
                      .code-sample-ul {color: red;}<\/p>\n
                        \n
                      • Item 1<\/li>\n
                      • Item 2<\/li>\n<\/ul>\n<\/div>\n
                        Not every property passes from a parent to its child elements. Browsers deem certain properties as non-inherited<\/em> properties. Margins are one example of a property that isn\u2019t passed down from a parent to a child element.<\/p>\n
                        Specificity<\/span><\/h3>\n
                        The second rule that determines which rules are applied to each HTML element is the rule of specificity.<\/p>\n
                        CSS rules with more specific selectors<\/em> will overrule CSS rules with less specific selectors<\/em> regardless of which occurs first. As we discussed, the three most common selectors are element tags, classes, and ids.<\/p>\n
                          \n
                        • The least specific type of selector is the element level selector.<\/li>\n
                        • When a class is used as a selector it will overrule CSS rules written with the element tag as the selector.<\/li>\n
                        • When an ID is used as a selector it will overrule the CSS rules written with element or class selectors.<\/li>\n<\/ul>\n
                          Another factor that influences specificity is the location where the CSS styles are written. Styles written inline with the style<\/code> attribute overrule styles written in an internal or external stylesheet.<\/p>\n
                          Another way to increase the specificity of a selector is to use a series of elements, classes, and IDs to pinpoint the element you want to address. For example, if you want to pinpoint unordered list items on a list with the class \u201cexample-list\u201d contained with a div<\/code> with the id \u201cexample-div\u201d you could use the following selector to create a selector with a high level of specificity.<\/p>\n
                          div#example-div > ul.example-list > li {styles here}\n<\/code><\/pre>\n
                          While this is one way to create a very specific selector, it is recommended to limit the use of these sorts of selectors since they do take more time to process than simpler selectors.<\/p>\n
                          Once you understand how inheritance and specificity work you will be able to pinpoint elements on a web page with a high degree of accuracy.<\/p>\n
                          What Can CSS Do?<\/span><\/h2>\n
                          A better question might be: \u201cWhat can\u2019t CSS do?\u201d<\/p>\n
                          CSS can be used to turn an HTML document into a professional, polished design. Here are a few of the things you can accomplish wish CSS:<\/p>\n
                            \n
                          • Create a flexible grid for designing fully responsive websites that render beautifully on any device.<\/li>\n
                          • Style everything from typography, to tables, to forms, and more.<\/li>\n
                          • Position elements on a web page relative to one another using properties such as float<\/code>, position<\/code>, overflow<\/code>, flex<\/code>, and box-sizing<\/code>.<\/li>\n
                          • Add background images to any element.<\/li>\n
                          • Create shapes, interactions, and animations.<\/li>\n<\/ul>\n
                            These concepts and techniques go beyond the scope of this introductory guide, but the following resources will help you tackle these more advanced topics.<\/p>\n
                            The Box Model<\/span><\/h3>\n
                            If you plan to use CSS to build web page layouts, one of the first topics to master is the box model. The box model is a way of visualizing and understanding how each item on a web page is a combination of content, padding, border, and margin.<\/p>\n
